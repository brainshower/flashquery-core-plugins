# Product Brain — Tag Vocabulary

Tags provide lightweight, cross-document filtering without requiring a rigid taxonomy. Skills should suggest from this list when tagging new documents. Creating a new tag should be a deliberate choice — only when a genuinely new dimension is needed that the existing vocabulary doesn't cover.

## Workflow — what needs to happen next

- `#priority` — Actively being worked on, or needs attention soon. The short list of things that matter right now.
- `#blocked` — Progress requires something external — a decision, a dependency, more information. Not abandoned, just waiting.
- `#revisit` — Not urgent now but genuinely worth returning to. Softer than priority; stronger than someday.
- `#someday` — Worth keeping, no timeline. Parked deliberately.
- `#needs-review` — Ready for a deliberate review pass — either by the Review Loop or the user.

## Handoff — readiness to move somewhere else

- `#needs-spec` — Enough research has accumulated to write a feature spec. Prompts the Draft skill.
- `#dev-ready` — Feature spec is complete and ready for development handoff.
- `#milestone` — Associated with a milestone brief. Apply to specs and work items that are in-scope for the current release push.

## Classification — what kind of content this is

- `#decision` — Contains or records a product decision — made or pending.
- `#user-feedback` — Sourced from user or community input — a request, a complaint, a pattern observed.
- `#bug` — Something is broken or behaving unexpectedly.
- `#technical-debt` — Known work that should be addressed eventually. Not a bug, not a feature.
- `#research-needed` — More investigation required before this can move forward.
- `#decision-needed` — A specific decision is required to unblock this.

## Source — where the content originated

- `#community` — Came from or is relevant to the community (Slack, GitHub discussions, Discord).
- `#web-research` — Sourced via web search, monitoring, or link capture.
- `#codebase` — Derived from analysis of the source repository.
